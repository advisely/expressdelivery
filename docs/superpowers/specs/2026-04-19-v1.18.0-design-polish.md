## 1. Problem Statement

User feedback after testing the v1.17.5 update file confirmed the Yahoo delete bug was fixed. Two new asks:

1. "I like the animation effect when an email is deleted, check elsewhere where we can introduce same effects/animations."
2. "When some emails are HTML or have images, the content shows in a generic box, beautify that boxes with maybe rounded edges, elevations, shadows or better designer contours — have an expert designer settle this."

## 2. Solution

### 2.1 Animation extension (Phase 2 → Sidebar + ComposeModal)

Apply the ThreadList enter/exit pattern to two more lists where users frequently add/remove items:

**Sidebar folders** (`src/components/Sidebar.tsx` + `Sidebar.module.css`):
- Track `enteringFolderIds: Set<string>` + `exitingFolderIds: Set<string>` + a `folderEnterTimersRef: useRef<Set<Timeout>>` for cleanup on unmount.
- New `flagFoldersAsEntering` callback: register IDs in entering set, schedule a 400ms cleanup setTimeout (also tracked in the ref).
- `refreshFolders` now diffs the incoming `folders:list` result against the current store BEFORE calling `setFolders`, then invokes `flagFoldersAsEntering(newcomers)`. Since every folder mutation handler (`handleCreateFolder`, `handleDeleteFolder`, `handleRenameFolder`, `handleReorderFolder`) calls `refreshFolders`, this single instrumentation point catches all "folder was created" cases.
- `handleDeleteFolder.doDelete` runs `Promise.all([ipcInvoke('folders:delete'), setTimeout(250)])` so the row visually exits in parallel with the IMAP delete. After the IPC + animation both complete, the row is removed by the `refreshFolders` call and the exiting set is cleared.
- CSS keyframes `nav-item-row-in` (slide-in-from-left, max-height grow) and `nav-item-row-out` (slide-out-right, max-height collapse + margin/padding zero), gated behind `prefers-reduced-motion: no-preference`.

**Compose attachment chips** (`src/components/ComposeModal.tsx` + `ComposeModal.module.css`):
- Track `enteringAttIds` + `exitingAttIds` + `attEnterTimersRef`.
- `handleAttachFiles` flags the new IDs as entering for 350ms after appending to state.
- `handleRemoveAttachment` flags the ID as exiting, then a 220ms `setTimeout` (tracked in the ref) actually removes it from the attachments array. Test updated to `await waitFor` for the now-animated removal.
- CSS keyframes `compose-chip-in` (scale 0.85 + translateY -4 → 1 + 0) and `compose-chip-out` (scale 1 → 0.85, max-width + margin + padding + border-width all collapsed to 0). Reduced-motion respected.

The ThreadList exit-animation test was tightened to use `waitFor` for the exit class to clear, eliminating cross-test pollution that had been flaking in the full vitest run.

### 2.2 Email body card design (ux-ui-architect spec)

Dispatched to the `ux-ui-architect` agent with the user's verbatim feedback. Spec returned, applied verbatim:

**New theme tokens (`src/index.css`)** — added to all 4 theme blocks:
- `--shadow-card-md`: per-theme shadow values calibrated for legibility on each background. Light + Cream get warm low-opacity shadows. Midnight + Forest get deeper darker shadows that actually read on dark backgrounds.
- `--email-surface-border`: derived from `--color-border-strong` with theme-tuned alpha (light themes 0.6-0.7, dark themes 0.55).

**`.email-iframe` (`ReadingPane.module.css`)** — the iframe IS the card now:
- 1px border using `--email-surface-border`
- `border-radius: 10px` (matches `.rp-popover` and stays in the codebase's 6-12px band)
- `background: var(--bg-elevated)` (slightly lifted from page bg in all 4 themes)
- `box-shadow: var(--shadow-card-md)`
- `transition: box-shadow 0.2s ease, border-color 0.2s ease` gated behind `@media (prefers-reduced-motion: no-preference)`

**`.plain-text-card` (new)** — the same elevated treatment for text-only emails (`pre` block with monospace font, 16px/24px padding inside the card).

**`buildIframeSrcdoc` body style (`ReadingPane.tsx`)** — bumped `padding` from `0` to `16px 20px` so author HTML breathes inside the new rounded card, and switched `background:#fff` → `background:transparent` so the parent's `--bg-elevated` shows through (preserves tonal continuity across themes — the iframe srcdoc cannot read parent CSS variables, so transparent + parent bg is the only correct fix). Bumped `setContentHeight(e.data.height + 16)` to `+ 32` to account for the new vertical padding.

**Plain-text fallback JSX** (`ReadingPane.tsx:1060`) swapped from inline `<div style={{ whiteSpace: 'pre-wrap' }}>` to `<pre className={styles['plain-text-card']}>` for parity.

### 2.3 Designer's explicit non-recommendations (followed)

The ux-ui-architect explicitly **declined** to inject `img { border-radius / box-shadow }` into the iframe srcdoc, reasoning that marketing emails (newsletters, receipts) frequently rely on hero images bleeding edge-to-edge or stitched image grids; rounding them would break the sender's intended composition. Image styling stays under sender control. Out of scope.

## 4. Risks

| Risk | Mitigation |
|---|---|
| Test flakiness from real-timer animations leaking between tests | All exit timers tracked in unmount-cleanup refs. Test cleanup uses `waitFor` on the absence of the exit class, guaranteeing the timer + state update have completed before the next test runs. |
| Iframe transparent background reveals page background in legacy themes | All 4 themes verified to have `--bg-elevated` distinct from `--bg-primary` (via the theme-token deltas in `src/index.css`). The iframe's parent wrapper provides the elevated bg, the iframe shows it through transparency. |
| Reduced-motion users miss the animation entirely | Acceptable per WCAG SC 2.3.3 — they get the data change without motion, which is the explicit user preference. |
| Folder reorder/rename triggers entry animation incorrectly | False — `refreshFolders` diffs ID sets, not full row identity. Same ID = no animation regardless of name/order/color change. |

## 5. Acceptance Criteria

**Functional:**
- Open any HTML email → body renders inside a card with rounded corners + shadow that's visible against the page background in all 4 themes.
- Open a plain-text-only email → text appears in a monospaced card with the same treatment.
- Sidebar → right-click a non-system folder → Delete: row slides out to the right + collapses height while the IMAP delete completes in parallel.
- Sidebar → create a new folder via context menu: row slides in from the left (~300ms).
- Compose → attach a file: chip scales-fades in. Click the X on a chip: chip scales-collapses out before disappearing.
- Enable Windows Settings → Accessibility → Visual effects → Animation effects: off → all animations skip; data changes happen instantly with no motion.

**Tests:**
- Vitest: 1075/1075 (no new tests in this round; existing tests updated for animated removal).
- ESLint: 0 warnings.
- TS strict: clean.

## 6. Out of Scope

- Image styling inside iframe srcdoc (rejected by designer; sender content is sender-controlled).
- Animation extension to tags / saved searches / mail rules / accounts (low-frequency CRUD; revisit if user requests).
- Card treatment for `AttachmentPreviewModal` body (separate component; no user complaint yet).
